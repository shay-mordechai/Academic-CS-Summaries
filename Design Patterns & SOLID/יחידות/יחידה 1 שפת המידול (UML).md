## יסודות UML

### יחידה 1: שפת המידול (UML)

הסיכום יתמקד בכלי המידול של מבנה (Class Diagrams) והתנהגות (Statecharts, Sequence Diagrams).

| **נושא**                                    | **נקודות ממוקדות לסיכום**                                                                                                                             |
| ------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[[תרשימי מחלקות (Class Diagrams)]]**      | **קשרים:** הגדרה ברורה של ההבדלים בין **Association**, **Aggregation**, ו-**Composition**.                                                            |
|                                             | **Composition** מייצג יחס "שלם-חלק" ותלות חיים, כאשר אובייקט החלק נוצר ונמחק יחד עם אובייקט השלם. **Aggregation** מייצג בעלות ללא תלות חיים.          |
|                                             | **ריבוי (Multiplicity):** הבנת המשמעות של סימוני ריבוי (`1..*`, `1`, `0..1`).                                                                         |
|                                             | **Generalization/Realization:** ירושה (Generalization) ומימוש ממשק (Realization).                                                                     |
| **[[תרשימי מצבים (Statecharts)]]**          | **מבנה:** הבנת המרכיבים העיקריים: מצבים (**States**), מעברים (**Transitions**), אירועים (**Triggers**), תנאי שמירה (**Guard**), ופעולות (**Action**). |
|                                             | **יישום:** הבנת סדר הפעולות במחברי תנאי.                                                                                                              |
|                                             | זכירת שכלולים כמו מעבר פנימי והיררכיית מצבים. תרגול ניתוח מצבים המופעלים על ידי אירועים עם ירושה.                                                     |
| **[[תרשימי רצף (Sequence+Communication)]]** | **תפקיד:** הבנה שתרשים רצף ותרשים תקשורת מתארים את אותו מידע (אינטראקציה דינמית) אך מתמקדים בייצוג שונה (רצף זמן מול קשרים בין אובייקטים).            |

---

המטרה כאן היא לא ללמוד הכל מההתחלה, אלא לרענן את המושגים המרכזיים שתצטרך כדי להבין ולענות על שאלות במבחן, במיוחד קריאת תרשימים קיימים.

### חזרה מהירה: קשרים בתרשים מחלקות (Class Diagram)

החלק הכי חשוב בתרשימי מחלקות הוא להבין את המשמעות של החצים. הנה טבלה שמסכמת את הקשרים המרכזיים שלמדנו:

|**הקשר (Relationship)**|**הסימון (Arrow)**|**מה זה אומר? (Meaning)**|**דוגמה שלנו**|
|---|---|---|---|
|**Association** (קישור)|`---->` (קו רציף, חץ פתוח)|`A` "מכיר" / "משתמש ב-" `B`|`Client` מכיר `Shape`|
|**Aggregation** (קיבוץ)|`<>---->` (מעוין פתוח)|`A` "מכיל" את `B` (יחסי שלם-חלק)|`Department` מכיל `Employees`|
|**Composition** (קומפוזיציה)|`<◆>---->` (מעוין מלא)|`A` "מורכב מ-" `B` (יחסי שלם-חלק חזקים)|`Drawing` מורכב מ-`Shapes`|
|**Generalization** (ירושה)|`---▷` (קו רציף, חץ משולש)|`A` הוא סוג של `B` (extends)|`RoadLogistics` הוא סוג של `Logistics`|
|**Realization** (מימוש)|`- - -▷` (קו מקווקו, חץ משולש)|`A` מממש את `B` (implements)|`Circle` מממש את `Shape`|
|**Dependency** (תלות)|`- - ->` (קו מקווקו, חץ פתוח)|`A` "תלוי ב-" `B` באופן רגעי|`ShapeFactory` יוצר `Shape`|
